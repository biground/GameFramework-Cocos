# 一致性检查流程

## 执行步骤

### 1. 读取基准文件
- 读取 `docs/module-registry.md` — 模块依赖和 API 基准
- 读取 `docs/consistency-guide.md` — 三维度检查标准
- 读取当前模块的 `README.md` — 模块设计契约

### 2. 架构一致性检查
- [ ] 所有模块继承 ModuleBase
- [ ] framework/ 层无 cc 命名空间引用
- [ ] 模块间通过 GameEntry.getModule() 获取引用
- [ ] 跨模块事件走 EventManager
- [ ] priority 分配符合规范

### 3. 风格一致性检查
- [ ] 命名规范统一
- [ ] JSDoc 格式统一
- [ ] 错误处理模式统一
- [ ] 生命周期方法签名统一：onInit() → onUpdate(dt) → onShutdown()

### 4. 设计一致性检查
- [ ] 新模块的 API 风格与已有模块一致
- [ ] 错误消息格式统一：`[模块名] 描述性消息`
- [ ] 同类模块的方法命名模式一致
